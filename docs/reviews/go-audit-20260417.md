## Pre-Existing Failures Investigated

### `backups/2026-04-16/` — mixed-package Go files break `go build ./...`

**Root cause:** The directory `backups/2026-04-16/` at the project root contains Go
source files from two different packages — `tui` (`app.go`, `monitor.go`, `styles.go`)
and `sysinfo` (`sysinfo.go`, `sysinfo_test.go`) — plus `uninstall.go`. Go's toolchain
tries to compile the directory as a single package and fails:

```

```

**Not fixed here** (out of scope; these are manual backup snapshots). Recommendation:
add `backups/` to `.gitignore` or rename files to remove the `.go` extension.

## Critical Issues (Must Fix)

### CRIT-1 — `internal/remover/log.go:20` — silenced `os.UserHomeDir()` error → bad path

```go
// BEFORE (broken on home-less environments)
func logPath() string {
    home, _ := os.UserHomeDir()   // error silenced; home="" on failure
    return filepath.Join(home, ".config", "gone", "operations.log")
}
```

If `os.UserHomeDir()` fails, `home` is the empty string and `logPath()` returns
`"/.config/gone/operations.log"` — a root-relative path. `AppendLog` then attempts
`os.MkdirAll("/.config/gone", 0o755)` (requires root) and `os.OpenFile` on the same
path. Every log write silently fails or corrupts root dirs on non-standard setups.

`internal/scanner/locations.go` already uses the correct pattern — fall back to
`os.Getenv("HOME")`. Fixed to match.

**Commit:** `fix(remover): fallback to $HOME when os.UserHomeDir fails in logPath`

### CRIT-2 — `internal/remover/trash.go:11` — wrong AppleScript string escape

```go
// BEFORE (backslash-quote is not a valid AppleScript escape)
escaped := strings.ReplaceAll(absPath, `"`, `\"`)
script := fmt.Sprintf(`tell application "Finder" to delete POSIX file "%s"`, escaped)
```

AppleScript does **not** support `\"` as an escape sequence inside string literals.
When a macOS filename contains a `"` character (legal on APFS/HFS+), the generated
script is syntactically invalid AppleScript. `osascript` returns a parse error and
the file is not trashed — silently, from the user's perspective (only `stderr` is
captured).

Fix: pass the path as an `argv` argument to `osascript` so no string embedding is
needed:

```applescript
on run argv
    tell application "Finder" to delete (POSIX file (item 1 of argv))
end run
```

This eliminates the interpolation entirely and handles all path characters correctly.

**Commit:** `fix(remover): use osascript argv to avoid invalid AppleScript escape`

## Minor Issues (Nice to Fix)

### MIN-1 — `internal/scanner/scanner.go:32` — outer error always nil

`Search()` returns `([]Match, error)` but the outer error is always `nil`. Walk
errors from `fastwalk.Walk` are discarded (`_ = ...`). Individual entry errors
inside the walk callback return `nil` to continue scanning — that's correct. But the
function-level error return is misleading (callers have no way to know the walk
failed entirely for a root). Low practical impact since partial results are useful for
a search UI.

### MIN-2 — `internal/sysinfo/sysinfo.go:71` — blocking 500 ms sleep in TakeSnapshot

The `time.Sleep(500 * time.Millisecond)` between CPU-baseline seeding and
measurement is intentional for correct CPU delta sampling. No goroutine leak.
However, since `TakeSnapshot` is called from the TUI's update loop, each refresh
blocks for 500 ms on the calling goroutine. No concurrency issue but relevant to UI
responsiveness. Out of scope to change (charmland-expert owns TUI scheduling).

### MIN-3 — `internal/remover/log.go` — `logPath()` called twice in `AppendLog`

`logPath()` → `os.UserHomeDir()` is called once for the `MkdirAll` and again for
`OpenFile`. Functionally harmless but the home dir is resolved twice. After CRIT-1 is
fixed this remains a minor efficiency note; no behavior change.
